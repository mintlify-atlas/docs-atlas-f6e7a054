---
title: "FlowKit — data-assets"
description: "A self-contained Python ETL library for extracting engineering data into PostgreSQL, orchestrated by Apache Airflow."
---

FlowKit (`data-assets`) is a Python library that gives Apache Airflow a single function call to run a complete ETL pipeline. Airflow decides **when** and **whether** to run. FlowKit handles **everything else**: extraction, rate limiting, checkpointing, schema management, validation, and atomic promotion.

```python
from data_assets import run_asset

result = run_asset("sonarqube_projects", run_mode="full")
# {'run_id': '...', 'rows_extracted': 42, 'rows_loaded': 42, 'duration_seconds': 3.2, 'status': 'success'}
```

## Key features

<CardGroup cols={2}>
  <Card title="Atomic ETL runs" icon="shield-check">
    Every run uses a temp table. Data is promoted to the main table only on success — no partial writes.
  </Card>
  <Card title="Resumable extraction" icon="rotate-right">
    Checkpoint-based retry. Failed runs resume from where they stopped — no wasted API calls.
  </Card>
  <Card title="Parallel extraction" icon="bolt">
    Page-parallel and entity-parallel thread pool modes share a single rate limiter across all workers.
  </Card>
  <Card title="Self-managing schemas" icon="table">
    Tables are auto-created and columns are added automatically as your asset definition evolves.
  </Card>
  <Card title="Rate limiting built-in" icon="gauge">
    Thread-safe sliding-window rate limiter keeps API calls under your configured limit across all workers.
  </Card>
  <Card title="Stale-run recovery" icon="clock-rotate-left">
    If a worker dies, the next run detects the abandoned run, inherits its temp table, and resumes.
  </Card>
</CardGroup>

| **Jira** | projects, issues |
| **Transforms** | Postgres-to-Postgres derived tables |

## Requirements

- Python 3.11+
- PostgreSQL 14+
- Apache Airflow 2.x+ (for scheduling)
