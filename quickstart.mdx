---
title: "Quick start"
description: "Install data-assets and run your first ETL asset in under 5 minutes."
---

<Steps>
  <Step title="Install the package">
    Install `data-assets` from PyPI:

    ```bash
    pip install data-assets
    ```

    To include development and testing dependencies:

    ```bash
    pip install data-assets[dev]
    ```
  </Step>

  <Step title="Set up PostgreSQL">
    Create a database and make sure the connecting user has `CREATE SCHEMA` privileges:

    ```sql
    CREATE DATABASE data_assets;
    ```

    You do not need to create any schemas or tables manually. On first run, FlowKit auto-creates the required schemas (`data_ops`, `raw`, `mart`, `temp_store`) and all metadata tables.
  </Step>

  <Step title="Configure credentials">
    Set the `DATABASE_URL` environment variable. This is the only credential required to run any asset:

    ```bash
    export DATABASE_URL="postgresql://user:pass@localhost:5432/data_assets"
    ```

    Each data source has its own variables. For SonarQube:

    ```bash
    export SONARQUBE_URL="https://sonar.example.com"
    export SONARQUBE_TOKEN="sqa_xxxxx"
    ```

    <Note>
      For the full list of per-source environment variables, see the [configuration guide](/guides/configuration).
    </Note>
  </Step>

  <Step title="Run your first asset">
    Call `run_asset()` with the asset name and a run mode:

    ```python
    from data_assets import run_asset

    result = run_asset("sonarqube_projects", run_mode="full")
    print(result)
    ```

    A successful run returns a dict with run metrics:

    ```python
    {
        'run_id': '01927f3a-...',
        'asset_name': 'sonarqube_projects',
        'partition_key': '',
        'rows_extracted': 42,
        'rows_loaded': 42,
        'duration_seconds': 3.2,
        'status': 'success',
        'metadata': { ... }
    }
    ```

    <Note>
      `run_mode` controls what data is fetched. The four modes are:

      | Mode | When to use |
      |------|-------------|
      | `full` | Initial load or periodic full refresh |
      | `forward` | Incremental — fetch new data since the last run |
      | `backfill` | Fill in historical data going backwards |
      | `transform` | Run Postgres-to-Postgres SQL transforms |

      See the [run modes guide](/guides/run-modes) for details.
    </Note>
  </Step>

  <Step title="Deploy with Airflow">
    For production scheduling, copy `example_dags/flowkit_dags.py` to your Airflow DAGs folder. Each asset gets its own DAG with retries and staggered schedules pre-configured.

    For auto-discovery of all registered assets, use the DAG factory instead:

    ```python
    # dags/data_assets_all.py
    from example_dags.dag_factory import create_dags

    globals().update(create_dags())
    ```

    `create_dags()` inspects the asset registry and generates one DAG per asset (or one DAG per org for GitHub assets). It reads secrets from Airflow Connections at execution time — no pre-configured environment variables needed on workers.
  </Step>
</Steps>
